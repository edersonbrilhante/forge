# Splunk Dashboard Panel Reference

This page lists the Forge Splunk dashboards and the panels operators should
expect to see. Use it with the [Splunk Dashboard Runbook](splunk-dashboard-runbook.md).

The exact SPL and SignalFlow live in the Terraform dashboard definitions:

- Splunk Cloud: `modules/integrations/splunk_cloud_conf_shared`
- Splunk Observability: `modules/integrations/splunk_o11y_conf_shared`

## Panel Reading Rules

- Start with freshness before severity.
- Prefer tenant, repository, runner group, region, and time-window filters before
  reading a table as a platform-wide signal.
- Treat failure counts without denominator as suspicious. Compare failures to
  total events, jobs, runners, or tenants.
- Treat missing `forgecicd_tenant` as a data-quality issue unless the sourcetype
  is intentionally platform-wide.
- Use trends for broad health and tables for ownership or drilldown.
- Use Splunk Cloud logs to explain what happened, and Splunk Observability
  metrics to explain resource pressure.
